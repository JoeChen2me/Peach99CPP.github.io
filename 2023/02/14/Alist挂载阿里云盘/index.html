Alist挂载阿里云盘

概览

​ 在2023-02-13日开始，阿里云盘官方开始封禁第三方API，导致之前主流的AList、 小白羊、蜗牛云盘等第三方播放工具都失效了。本人一开始也是使用的Alist挂载阿里云盘作为影视NAS使用，也受到了本次封禁的影响，出现了报错提示。

ailed get link: invalid X-Device-Id

​ 在经过一段时间的尝试后，发现通过Alits挂载阿里云盘的分享链接的方式不受本次的影响。
因此以下为通过分享链接的方式进行挂载的教程

配置AlistV3

  • 安装Alist

    • Github Release

    • 官方文档

    • 在Release界面下载zip格式的Alist v3，解压后进入文件夹

    • 进入cmd运行如下指令

    • 在浏览器打开127.0.0.1:5244 ，账号:admin 密码为上述提到的

    • 进入页面下方的管理界面

配置Alist链接

参照官方文档

  • 需要用到的东西
    • token
    • 分享链接
  • 在管理界面点击存储，进行添加

  • 配置存储器路径 需要记住此处的挂载路径

  • 根据分享链接进行信息填写

    • 刷新令牌即为你自己的阿里云盘的Token
    • 分享ID来自于分享链接
    • 根文件ID也来自于分享链接
    • 附图是官方文档中关于如何从分享链接中解析出上述信息的解释

  • 完成上述填写后点击保存，而后回到127.0.0.1:5244进行验证
    进入路径查看是否能获取到数据
    如图所示 可以正确获取到链接中的数据

配置RaiDrive

进入RaiDrive 点击Add

选择NAS中的WebDAV选项

填写配置如下图

  • 上方绿色部分为驱动器在本机显示出的路径，可以较为随意，只要不与已有的盘符冲突即可
  • 红色部分中127.0.0.1:5244是Alist的地址，照着填写即可。此处注意需要取消前方Address的勾选
  • 黄色部分，其中/dav是默认的，/F与上述配置Alist中所分配的驱动器路径相对应，我填写的是F
  • 下方绿色部分为账号密码，即为一开始在网页端登录Alist输入的账号密码
  • 上述输入完成后点击connect即可链接到云盘内容，此处需要注意确保Alist处在运行状态

验证

  1. 查看网络位置中的驱动器

  2. 进入文件夹，能获取到链接内数据

  3. 验证文件传输，选择的文件如下：狂飙第23集 4K版本
    双击播放该文件，在播放器中查看读取到的文件的属性
    与文件原属性相对应，因此可以实现原画播放
